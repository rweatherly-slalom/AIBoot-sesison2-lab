# TODO App Functional Requirements

## Core Task Management

1. The user can create a new task with a required title.
2. The user can edit an existing task title.
3. The user can delete a task.
4. The user can mark a task as completed or not completed.

## Task Details

5. The user can add an optional description to a task.
6. The user can add or update an optional due date for a task.
7. The user can assign a priority level to a task (for example: low, medium, high).

## Organization and Visibility

8. The user can view tasks in separate groups: all, active, and completed.
9. The user can sort tasks by a selected order, including:
   - due date (earliest first)
   - creation date (newest first)
   - priority (high to low)
10. The user can search tasks by title or description text.

## Data Persistence

11. Task data persists between sessions so tasks remain after the app is closed and reopened.

## Validation and Usability

12. The app prevents creating a task with an empty title.
13. The app clearly indicates overdue tasks when a due date is in the past and the task is not completed.
14. Completed tasks remain visible unless the user filters them out.
